## Texture Cache Relocation

Extended `src/TextureManager/TextureFileCache.cs` so oversized replacements no longer fail in-place.

Implemented:

- build all replacement mip payloads first
- if in-place replacement fits, overwrite original mip ranges
- if replacement exceeds old allocation, append new mip chain to end of `.tfc`
- update manifest mip offsets and sizes to relocated data
- preserve existing backup behavior

Result:

- texture injection can now succeed even when replacement data is larger than original cache allocation

## Runtime Safety Guard

Added exact dimension enforcement in `src/TexturePreview/Services/TexturePreviewInjector.cs`.

Implemented:

- reject injection unless source width and height exactly match target `Texture2D` `SizeX` and `SizeY`
- clear validation message reports both source and target dimensions

Reason:

- current texture path updates cache data and manifest offsets only
- it does not rewrite UPK `Texture2D` size metadata
- mismatched dimensions are unsafe for runtime use
